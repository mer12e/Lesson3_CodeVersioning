Highlight #1 Version Control

Version control is the management of changes to documents, computer programs, large web sites, and other colletions of information.It consists of one main repository for all of the project files. With version control, team members can check files out, make changes, then check them back in (or commit them).

Highlight #2 Versioning Control System

A versioning control system, or VCS, is a system that records changes to a file or set of files over time so that you can recall specific versions later. The system automatically notes who changed the files (revision history), when they were changed, what about them was new or different, and merge changes to the same file. A VCS has many advantages, such as that files cannot be overwritten and people can work on the same files simultaneously without conflict. Furtermore, there are two types of repositories within a VCS, including a remote repository and a local repository.

Highlight #3 GIT

GIT is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency. It is very distributed. GIT is beneficial to use because every user has a complete copy of the repository data stored locally on their machine, everything is local, there is no single point of failure, processes run much faster, and GIT will prevent an unintentional overwrite of a file on the shared repository. The three basic steps with GIT are to add, then to commit, and then to push.
